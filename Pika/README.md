# Take-Home Assignment — Senior iOS Engineer

You'll receive a Figma link with design mocks for a focused iOS app.

Your job is to implement the flow with these mocks as your source of truth. This brief is intentionally non-exhaustive — what you do with the gaps is part of what we're evaluating.

## Principles

- **Care about designs.** The designer wants to see their work handled with care.
- **State permutations and edge cases aren't covered in the mocks.** Fill them in. Use your judgment.
- **This codebase will grow.** Write things you'd be happy to find when you return to them in six months, or hand off to another senior engineer without a walkthrough.
- **There's a backend engineer — but they're busy.** Any API requirements should be architected as if a real implementation will slot in later. The seam should be obvious and clean. The UI should still work today.
- **Assets matter.** Think about what the app needs and how you'd best structure it.
- **Production ready means maintainable.** Tests and testability are not required. But another senior engineer should be able to pick this up and ship it without a handover call.
- **Use SwiftUI.** UIKit interop is fine where genuinely necessary, but SwiftUI is the foundation.
- **AI assistants are encouraged.** Use whatever tools help you work well. We're evaluating the output and the decisions behind it, not the process.
- **You're in a team.** Think about and document any questions or suggestions you would have to your imaginary designer or backend engineer counterpart.

## Time

3–5 hours. Use your judgment on where to invest your time.

## Deliverable

Xcode project + a short README covering your architecture decisions, anything you'd revisit with more time, and any questions or suggestions you'd bring to the designer and backend engineer before or during implementation.
